Class ClientException

Description
Description | Vars | Methods

Thrown by all IClient methods in case an error takes place and method cannot return its results normally.

You should always put your IClient methods into try/catch blocks and make sure you catch this exception. If you catch this exception, the message borne will be adequate to explain to you what has gone wrong. Especially the message borne by the InnerException of the Exception caught.

Here is how you can wrap your code arround try/catch block and on error get information about what has gone wrong:

  1.  try {
  2.   $event = new \Fraudpointer\API\Models\Event(\Fraudpointer\API\Models\Event::$FAILED_PAYMENT);
  3.   $event->AddData("MERCHANT_REFERENCE", GetAcmeOrderNumber());
  4.  
  5.   // PURCHASE_DATE
  6.   $event->AddDateValue("PURCHASE_DATE", time());
  7.  
  8.   $event_returned = $client->AppendEventToAssessmentSession($assessment_session, $event);
  9.   return $event_returned;
  10.  }
  11.  catch (\Fraudpointer\API\ClientException $e) {
  12.  }
