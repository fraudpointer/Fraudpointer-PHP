  100.    /**
  101.      * Use this method to generate a hash of a credit card number. Use the generated hash to send the encrypted credit card number to FraudPointer Server instead of the credit card number itself. <br/>
  102.      * <br/>
  103.      * FraudPointer Server tries to identify the existence of the same credit card number in various transactions. These
  104.      * transactions either take place during the same session or take place in different sessions, but at the same time, or
  105.      * took place in a session in the past. However, FraudPointer Server does not want to store the credit card numbers in clear format
  106.      * and it does not need to do that in order to accomplish its goal. Hence, you need to encrypt them using the method provided here.
  107.      * Note that hash is one-way encryption method and FraudPointer Server cannot derive the credit card number from the hash.
  108.      * <br/>
  109.      * This method does not communicate with the FraudPointer Server to generate the hash. Works locally.<br/>
  110.      * <br/>
  111.      * @param string $credit_card_number The credit card number that you want to get its hash value.
  112.      * @return string The hash of the credit card number. Note that if an empty string is returned, then something has gone wrong during the encryption.
  113.      *  in that case, method throws a ClientException.
  114.      */
  115.     public function CreditCardHash($credit_card_number);
