Class Kase

Description
Description | Vars (details)

A Case is created for every final FraudAssessment.

The Case is meant to be handled by a human interacting with the Fraudpointer Application. When a Case is created, it has an "Open" status (unless you have rules in your Fraudpointer application configuration that automatically put them in other statuses).

When a Case is inspected by a human, human might decide to set a Kase::resolution to it.

Note: We use the class identifier "Kase" instead of "Case" because php compiler had problem with this.
