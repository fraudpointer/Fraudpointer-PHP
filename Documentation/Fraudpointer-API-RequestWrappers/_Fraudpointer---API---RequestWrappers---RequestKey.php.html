Class Description
RequestKey Whatever we send over to Fraudpointer Server needs to have the api key. So Event and FraudAssessment need to be wrapped before serialization to json format and sent over with the api key. And this class is the parent class that will hold the api key part.

Note:ClientFactory::Construct() method returns an object that implements IClient interface and hides all the complexity of sending to and receiving data from Fraudpointer Server. If you use this object, you will not be worry about internals of communications.
